# Internationalization (i18n) contract

Favilla ships multilingual: **Italian (`it`, canonical source)** plus **English (`en`), French (`fr`), German (`de`), Spanish (`es`)**. Italian is authored first; the others are translations of it. This doc is the contract for adding/maintaining translatable copy.

## Non-negotiables

- **No hardcoded user-facing copy.** Every user-facing string goes through `t()` (or an overlay, see below). Italian is just another value file — never special-cased in code.
- **Keys are English, symbolic, dot-notation:** `t('contacts.form.label.company')`, `t('common.action.save')`. The first segment is the lang-file namespace; the rest is the nested path. Matches the project's "English structure / localized content" philosophy.
- **Always escape on output.** `t()` returns untrusted-by-convention text: wrap with `e()` when echoing into HTML — `e(t('contacts.title'))`. The only exception is developer-authored strings that intentionally contain markup (e.g. `errors.*` messages with `<br>`), echoed raw.
- **After adding/removing keys** run `php favilla lang:check` — it must pass (exit 0) before merge.

## Layout

```
resources/lang/<locale>/<namespace>.php   -> returns a (possibly nested) array
```

Namespaces: one per module (`contacts.php`, `tasks.php`, …) plus cross-cutting `common`, `validation`, `errors`, `auth`, `nav`, `datetime`, and the slug-keyed overlays `permissions`, `admin_panel`, `notifications`.

## Helpers (`app/Helpers/functions.php`)

| Helper | Use |
| --- | --- |
| `t($key, $replace = [], $locale = null)` | Translate a dot-key. Returns the key unchanged on miss (never fatals). |
| `__($key, …)` | Alias of `t()`. |
| `tc($key, $n, $replace = [], $locale = null)` | Pluralized: lang value is `"singular\|plural"`; `:count` auto-filled. |
| `t_line($namespace, $key, $default = '', $locale = null)` | **Flat** lookup for overlays whose keys contain dots (slugs/route-ids). Falls back to `$default` (the canonical Italian from `module.json`). |
| `locale()` / `set_locale($code)` | Read / set the active locale. |
| `format_date($dt, $mode, $locale = null)` | Locale-aware date. Modes: `time\|short\|compact\|relative\|long`. Numeric layout stays `dd/mm/yyyy`; month/day/relative labels are localized from `datetime.php`. |
| `format_number($v, $decimals, $locale)` / `format_currency($v, $cur, $locale)` | Locale-aware numerics (ext-intl, with EU fallback). |
| `format_date_it($dt, $mode)` | Back-compat shim → `format_date($dt, $mode, 'it')`. Prefer `format_date()` in new code. |

Placeholders use `:name` (Validator style) and `{{name}}` (mail-template style): `t('validation.required', ['field' => 'Email'])`.

## How locale is resolved

Resolved **once per request** in `bootstrap/app.php` (after the session starts — the core `Application` has no global-middleware slot) by `App\Services\LocaleResolver`, priority:

1. `?lang=xx` query param (persists to session + cookie)
2. `$_SESSION['user_language']` (primed at login from the stored preference)
3. `favilla_lang` cookie
4. logged-in user's `user_preferences.language` (heals sessions predating the feature)
5. `Accept-Language` header
6. `config('localization.default')` (`it`)

Switcher: public `GET /lang/{code}` (`Auth\LocaleController`, works anonymous + authenticated), surfaced by `app/Views/partials/language_switcher.php` in the header and auth layout. `<html lang>` is emitted from `$currentLocale`.

## module.json overlays (config-driven strings)

`module.json` (`navigation`/`menu`, `permissions`, `admin_panel`, `notification_events`) **stays Italian** — it feeds `context:generate`, `project_context.json`, and DB seeds; rewriting it to keys would corrupt all of that. Instead, translate **at render time** via slug-keyed overlay files, with the `module.json` value as fallback:

- Nav labels — `NavigationRegistry::normalizeEntry()` → `t_line('nav', $id, $jsonLabel)`.
- Permission labels — roles UI → `t_line('permissions', $slug, $dbName)`. DB `permissions.name` stays Italian (admin-canonical).
- Module descriptions (Admin › Modules list + uninstall confirm, non-core/auto-discovered modules only) — `t_line('modules', $moduleDirName, $jsonDescription)`.

## Adding a locale

1. Append the code to `supported` + `names` + `intl_locale` + `currency` in `app/Config/localization.php`.
2. Create `resources/lang/<code>/*.php` mirroring the `it` files (and `datetime.php`).
3. `php favilla lang:check --locale=<code>` until clean.

## Adding strings to a module (the Contacts pattern)

> **New modules start multilingual.** `make:module` scaffolds `t()`-based stubs **and** `resources/lang/{it,en,fr,de,es}/<module>.php` (Italian canonical + copies to translate). The `_Template` example module (`app/Modules/_Template`, namespace `example`) is the worked multilingual reference. For a freshly scaffolded module just translate the generated copies and run `lang:check`. The steps below are for **retrofitting a pre-existing** hardcoded module.

`Contacts` is the worked reference (`resources/lang/*/contacts.php`, `app/Modules/Contacts/Views/index.php`, `ContactsController.php`):

1. Grep hardcoded Italian in the module's `Views/` + `Controllers/` + `Services/`.
2. Author `resources/lang/it/<module>.php` with the verbatim Italian under English keys.
3. Replace literals with `t('<module>.…')`, **keeping `e()`** where output was escaped.
4. Field-label arrays become `t('<module>.fields.<field>')` (build the map by looping `array_keys($rules)`).
5. Translate into en/fr/de/es → `php favilla lang:check` → visual QA per locale.

## Tooling

```bash
php favilla lang:check                 # all locales vs the `it` baseline (exit 1 on missing keys)
php favilla lang:check --locale=de     # one locale
php favilla lang:check --strict        # also fail on EXTRA / EMPTY keys
```

## Caveats

- Machine-assisted translations: a **native-speaker review pass** (especially `de`/`fr`) is advised before public release.
- Existing DB `mail_templates` rows are seeded from `module.json` defaults and are **not** re-seeded by editing `module.json`; localized notification delivery resolves per-recipient with Italian fallback.
